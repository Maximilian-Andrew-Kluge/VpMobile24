Installation

In 3 Schritten startklar

Wähle deine bevorzugte Installationsmethode. HACS ist der empfohlene Weg.

1

HACS öffnen & Repository hinzufügen

Wähle eine der folgenden Methoden um VpMobile24 in HACS hinzuzufügen:

⭐ Option A – Direkt in HACS öffnen

Klicke auf den Button und HACS öffnet sich automatisch mit dem vorausgefüllten Repository:

🏠 In HACS öffnen

Empfohlen – funktioniert nur wenn my.home-assistant.io in deiner HA-Instanz konfiguriert ist.

🔗 Option B – Repository URL manuell

Öffne HACS, klicke auf ⋮ → „Benutzerdefiniertes Repository" und füge diese URL ein:

Repository URL
https://github.com/Maximilian-Andrew-Kluge/VpMobile24

Kategorie: Integration → Hinzufügen klicken.

2

Integration installieren & HA neu starten

Suche in HACS nach VpMobile24, klicke auf Installieren und starte Home Assistant anschließend neu.

💡 Nach dem Neustart erscheint die Integration automatisch in der Integrationsliste.
3

Integration einrichten

Gehe zu Einstellungen → Geräte & Dienste → Integration hinzufügen und suche nach VpMobile24.

Schul-ID Die ID deiner Schule (z.B. 10001329)
Benutzername Dein VpMobile24-Benutzername
Passwort Dein VpMobile24-Passwort
Klasse Klassenname, z.B. 10b
1

Integration herunterladen

Wähle eine der folgenden Methoden:

Option A – Git Clone

Klone das komplette Repository mit Git:

Terminal
git clone https://github.com/Maximilian-Andrew-Kluge/VpMobile24.git

Danach liegt der Ordner VpMobile24/custom_components/vpmobile24/ lokal vor.

Option B – ZIP herunterladen

Lade nur den custom_components-Ordner als ZIP herunter — immer die neueste Version:

⬇️ custom_components herunterladen

ZIP enthält den kompletten Repo-Stand. Entpacke und kopiere nur den Ordner custom_components/vpmobile24/.

2

Dateien kopieren

Kopiere den Ordner custom_components/vpmobile24/ in dein Home Assistant Konfigurationsverzeichnis:

Zielpfad
/config/custom_components/vpmobile24/
3

Home Assistant neu starten & einrichten

Starte Home Assistant neu. Gehe dann zu Einstellungen → Geräte & Dienste und füge die VpMobile24 Integration hinzu.

Lovelace Card

Die Stundenplan Card einrichten

Die Card wird automatisch mit der Integration installiert. Du musst nur noch die Ressource registrieren.

1

Ressource in Home Assistant hinzufügen

Gehe zu Einstellungen → Dashboards, klicke oben rechts auf ⋮ und wähle Ressourcen.

Klicke auf + Ressource hinzufügen und trage folgendes ein:

URL
/local/vpmobile24/vpmobile24-card.js
Ressourcentyp JavaScript-Modul
💡 Die Card-Datei wird beim Start von Home Assistant automatisch nach /config/www/vpmobile24/ kopiert. Du musst nichts manuell kopieren!
2

Card zum Dashboard hinzufügen

Öffne dein Dashboard, klicke auf Bearbeiten → + Karte hinzufügen und suche nach VpMobile24 Card.

Alternativ kannst du die Card direkt per YAML hinzufügen:

Vollständige YAML-Konfiguration
type: custom:vpmobile24-card
entity: sensor.vpmobile24_week_table
additional_info_entity: sensor.vpmobile24_zusatzinfos
reload_entity: button.vpmobile24_reload
show_header: true
header_settings:
  title: Stundenplan
  class_name: 10b
highlight_today: true
show_time: true
use_custom_times: false
3

Card konfigurieren

Die Card bietet eine vollständige visuelle Konfiguration – kein YAML nötig. Alle Optionen im Überblick:

Troubleshooting

Etwas funktioniert nicht? Hier sind die häufigsten Probleme und ihre Lösungen.

Card wird nicht angezeigt
  1. Überprüfe, ob die VpMobile24 Integration installiert ist
  2. Überprüfe, ob die Ressource korrekt hinzugefügt wurde (/local/vpmobile24/vpmobile24-card.js)
  3. Leere den Browser-Cache mit Strg + F5
  4. Starte Home Assistant neu
Änderungen werden nicht übernommen
  1. Leere den Browser-Cache mit Strg + F5
  2. Öffne die Browser-Konsole (F12) und prüfe auf Fehler
  3. Stelle sicher, dass die neueste Version geladen wurde
Entity nicht gefunden / Sensor fehlt

Stelle sicher, dass die VpMobile24 Integration korrekt installiert ist und der Sensor sensor.vpmobile24_week_table existiert. Gehe zu Einstellungen → Geräte & Dienste und prüfe den Status der Integration.

404-Fehler bei der Ressource
  1. Stelle sicher, dass die Integration installiert und geladen ist
  2. Starte Home Assistant neu – die Card wird beim Start automatisch kopiert
  3. Prüfe die Logs unter Einstellungen → System → Protokolle
Stundenplan wird nicht aktualisiert

Die Integration ruft Daten alle 15 Minuten ab. Prüfe deine Zugangsdaten und stelle sicher, dass deine Schule VpMobile24 / Stundenplan24 verwendet. Bei anhaltenden Problemen erstelle ein Issue auf GitHub.

Pausen-Zeiten werden nicht übernommen

Wenn du eigene Pausenzeiten verwenden möchtest, musst du nach dem Aktivieren von „Eigene Uhrzeiten verwenden" die gewünschten Zeiten im aufklappbaren Bereich „Uhrzeiten-Anpassung" eintragen.

  1. Aktiviere die Option „Eigene Uhrzeiten verwenden" in der Card-Konfiguration
  2. Klappe den Abschnitt „Uhrzeiten-Anpassung" auf – er erscheint direkt darunter
  3. Trage dort deine gewünschten Pausen-Zeiten ein und speichere die Konfiguration
💡 Die Zeiten werden erst übernommen, wenn sie im aufgeklappten Bereich eingetragen und gespeichert wurden.
Aktuelle Stunde wird nicht grün markiert

Die grüne Hervorhebung funktioniert nur wenn show_time: true gesetzt ist und die Uhrzeiten korrekt konfiguriert sind. Die Card liest die Stunden-Zeiten und vergleicht sie mit der aktuellen Uhrzeit.

  1. Stelle sicher dass „Uhrzeiten anzeigen" aktiviert ist
  2. Bei eigenen Zeiten: prüfe ob die Zeiten im Format 07:45-08:30 eingetragen sind
  3. Die Markierung erscheint nur am heutigen Tag
Mehrere Klassen mit gleichen Zugangsdaten einrichten

Ab v2.4.9 kannst du mehrere Klassen derselben Schule mit identischen Zugangsdaten einrichten:

  1. Gehe zu Einstellungen → Geräte & Dienste → Integration hinzufügen
  2. Wähle erneut VpMobile24 und gib dieselben Zugangsdaten ein
  3. Trage eine andere Klasse ein (z.B. 10b statt 10a)
  4. Jede Klasse bekommt ein eigenes Gerät: VpMobile24 – 10a (12345) und VpMobile24 – 10b (12345)
💡 Gleiche Schule + gleiche Klasse wird automatisch verhindert – du bekommst eine Fehlermeldung wenn du versuchst eine bereits konfigurierte Kombination erneut anzulegen.
Ausfallstunden werden grau statt rot angezeigt

Ausfallstunden (Fach = --- oder leer) werden als rote Kacheln angezeigt. Falls sie grau erscheinen, liegt es an der Datenquelle – der Sensor liefert keinen Ausfall-Marker. Stelle sicher dass die Integration auf v2.4.9 aktualisiert ist.

ⓘ-Button wird nicht angezeigt

Der ⓘ-Button erscheint nur wenn additional_info_entity in der Card-Konfiguration gesetzt ist:

YAML
additional_info_entity: sensor.vpmobile24_zusatzinfos
